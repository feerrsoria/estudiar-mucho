# This is NOT the Next.js you know

This version has breaking changes — APIs, conventions, and file structure may all differ from your training data. Read the relevant guide in `node_modules/next/dist/docs/` before writing any code. Heed deprecation notices.
The development has to be TDD. It means that for each functionality a test will be written first and the functionality will be tested against it until it passes all tests.

The architecture will be idiomatic, but domain first design.

The usage of interfaces to decouple services and infrastructure will be encouraged, always if possible.

The files must be short, concise, with a clear separation of concerns.
